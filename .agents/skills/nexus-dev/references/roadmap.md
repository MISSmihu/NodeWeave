# 功能路线图

所有 NEXUS 功能模块的编号、归属与依赖。**新建功能前先查这里**，避免重复编号，并按依赖顺序实施。

## 编号规则

`F-XXX`：功能编号。按领域分段：
- `F-0XX`：基础设施（认证、用户、API 框架、合规）
- `F-1XX`：核心内容（帖子、博客、评论、附件）
- `F-2XX`：社区经济（论坛币、交易、签到、徽章）
- `F-3XX`：用户体系（等级组、头像、成就、个人中心）
- `F-4XX`：社区治理（板块、审核、举报、AI审核）
- `F-5XX`：账号安全（换绑、改密、找回、注销）
- `F-9XX`：体验增强（主题、PWA、无障碍）

## F-0XX 基础设施

### F-001：基础认证（邮箱注册/验证/登录/登出）
**优先级**：P0（所有功能的地基）  
**含**：注册时**勾选同意《用户协议》《隐私政策》**（未勾不通过注册）、邮箱验证、Turnstile 全员防刷  
**前端**：`login.html`（登录/注册 Tab + 协议勾选框 + Turnstile）  
**后端**：`api/auth.js`  
**依赖**：F-011（合规文档必须先有）  
**规范**：详见 `auth.md`、`compliance.md`  
**状态**：📋 待实施

### F-002：邀请码注册 🔑 **P0·注册门槛**
**优先级**：P0  
**说明**：**管理员后台可控开关**。开启后，邮箱注册必须填有效邀请码。  
**机制**：
- 管理员/版主可生成邀请码（批量、设有效期、设使用次数）
- 用户达到 Lv3+ 可自助生成 1 个邀请码（每月刷新 1 个名额）
- 邀请码与注册用户绑定（`referred_by`），被邀请人升级邀请人 +10 声望
- 邀请码格式：`NX-XXXXX-XXXXX`（8位 base32，带前缀防误输）
**数据表**：`invite_codes(code, created_by, max_uses, used_count, expires_at, status)`  
**开关**：`site_config.invite_code_required`（见 `admin-config.md`）  
**前端**：注册表单加邀请码输入框（开关开启时显示）；后台邀请码管理页  
**后端**：`api/auth.js` 注册校验 + `api/admin/invite-codes.js`  
**状态**：📋 待实施

### F-003：实名制认证 🔒 **P1·合规**
**优先级**：P1  
**说明**：**管理员后台可控开关**。三种模式可选：
| 模式 | 行为 |
|------|------|
| `off` | 不要求实名 |
| `optional` | 引导但不强制，实名后解锁部分功能（如发帖、悬赏） |
| `required` | 必须实名才能发帖/评论（注册可完成，但行为受限） |
**实现**：手机号 + 短信验证码（境内 SMS 服务商，如阿里云/腾讯云）  
**预留扩展**：身份证号字段（仅存 SHA-256 哈希，不存明文，未来对接公安二要素核验）  
**数据表**：`users` 加 `phone_verified`、`phone_hash`、`id_card_hash`、`real_name_status`  
**开关**：`site_config.real_name_mode`（见 `admin-config.md`）  
**前端**：实名认证页 `account/verify-identity.html`；未实名时限制提示  
**合规**：详见 `compliance.md`（实名信息最小化、不滥用）  
**状态**：📋 待实施

### F-004：第三方授权登录（OAuth） 🔑 **P1**
**优先级**：P1  
**支持供应商**：
- **GitHub**（开发者社区天然适配）
- **QQ**（国内用户主流，需腾讯开放平台申请）
- **Google**（海外用户，**需评估数据出境合规**）
**关键联检逻辑**（GitHub 特有）：
- 若开启「GitHub 注册年限审核」，调 GitHub API 取账号创建时间
- **注册满 1 年（可配置）的 GitHub 账号**：可免邀请码直接注册
- **不满 1 年的**：仍需邀请码（防小号刷注册）
**账号绑定**：一个三方账号只能绑一个 NEXUS 账号；老用户可在设置页绑定/解绑  
**前端**：登录页"三方登录"区 + 设置页绑定管理  
**后端**：`api/oauth.js`  
**规范**：详见 `oauth.md`（重定向流、各供应商差异、CSRF 防护）  
**开关**：每个供应商独立开关（见 `admin-config.md`）  
**状态**：📋 待实施

### F-011：法律合规文档与基础设施  🔒 **P0·必须先做**
**优先级**：P0（注册前必须有可勾选的协议）  
**含**：
- 《用户协议》《隐私政策》《社区规则》三份文档（中文，符合中国法律）
- ICP 备案信息展示（底部）
- 实名制预留接口
- 用户注销流程
- 未成年保护机制
**前端**：`legal/terms.html`、`legal/privacy.html`、`legal/rules.html`、`account/delete.html`  
**规范**：详见 `compliance.md`  
**状态**：📋 待实施

### F-012：管理员账号与超级权限
**优先级**：P0  
**说明**：首个管理员账号通过**数据库初始化脚本**创建（不通过注册接口），拥有全部权限。可指派其他管理员/版主。  
**后端**：`scripts/init-admin.sh`（一次性建站脚本，本地运行）  
**规范**：详见 `permissions.md`  
**状态**：📋 待实施

### F-013：站点配置中心（开关型功能总控）⚙️ **P0**
**优先级**：P0（邀请码、实名制、三方登录等都依赖它）  
**说明**：所有"管理员可开关"的功能统一存在 `site_config` 表，后台可视化配置，**改配置无需重新部署**。  
**管理的开关**（初始集合）：
| 配置项 | 说明 | 默认 |
|--------|------|------|
| `invite_code_required` | 邮箱注册是否需邀请码 | `false` |
| `real_name_mode` | 实名制模式 off/optional/required | `off` |
| `oauth_github_enabled` | GitHub 登录开关 | `false` |
| `oauth_qq_enabled` | QQ 登录开关 | `false` |
| `oauth_google_enabled` | Google 登录开关 | `false` |
| `github_age_threshold_days` | GitHub 免邀请码的年限阈值（天） | `365` |
| `github_age_bypass_invite` | 老 GitHub 账号是否免邀请码 | `true` |
| `email_verification_required` | 是否强制邮箱验证才能登录 | `true` |
| `ai_review_enabled` | AI 审核开关 | `false` |
| `registration_enabled` | 是否开放注册（维护期可关） | `true` |
| `signin_reward_enabled` | 签到奖励开关 | `true` |
**前端**：`admin/site-config.html`  
**后端**：`api/admin/site-config.js`（仅 admin/owner 可改）  
**规范**：详见 `admin-config.md`  
**状态**：📋 待实施

## F-1XX 核心内容

### F-101：论坛帖子系统
**优先级**：P1  
**含**：CRUD、分类（讨论/问答/分享）、标签、点赞、收藏  
**前端**：`editor.html`、`post.html`  
**后端**：`api/posts.js`

### F-102：博客文章系统
**优先级**：P1  
**含**：Markdown 长文、封面、系列  
**复用**：`editor.html`、`post.html`（加 `type` 区分）

### F-103：评论与回复系统
**优先级**：P1  
**含**：嵌套回复、楼中楼、点赞、最佳答案  
**后端**：`api/comments.js`

### F-104：帖子附件上传
**优先级**：P2  
**说明**：帖子/博客/回复可上传图片、文件。存 **Cloudflare R2**（免费 10GB），D1 只存元数据。  
**含**：
- 图片（jpg/png/gif/webp，单张 ≤5MB）
- 文档（pdf/zip/txt/docx，单文件 ≤20MB）
- 病毒/类型校验（Magic Number 校验，防伪装）
- 存储位置：R2 bucket `nexus-attachments`
**前端**：编辑器内嵌上传组件  
**后端**：`api/upload.js`、R2 binding  
**依赖**：F-101、F-103

### F-105：回复可见（隐藏内容）
**优先级**：P2  
**说明**：发帖者可标记部分内容为「回复可见」，用户评论后才解锁。  
**数据表**：`posts` 加 `hidden_content TEXT` 字段；`comments` 关联解锁  
**语法**：编辑器里用 `[hide]...[/hide]` 包裹  
**后端**：`/api/posts/:id` 返回时，对未回复用户隐藏 `hidden_content`  
**依赖**：F-103

### F-106：定时可见/隐藏
**优先级**：P2  
**说明**：内容可设置「定时可见」（到点才公开）或「定时隐藏」（到点自动隐藏）。靠 **Cloudflare Cron Triggers** 定时扫描切换状态。  
**数据表**：`posts` 加 `visible_at INTEGER`（定时可见）、`auto_hide_at INTEGER`（定时隐藏）  
**后端**：`api/cron.js`（每分钟触发），Cron Trigger 配置进 `wrangler.toml`  
**依赖**：F-101

## F-2XX 社区经济

### F-201：论坛币系统
**优先级**：P1  
**含**：余额（`users.coins`）、流水（`coin_transactions`）、打赏、悬赏  
**获取规则**：注册 +50、每日签到 +5（递增）、发精华帖 +20、被点赞 +1、被采纳答案 +20  
**数据表**：`coin_transactions(id, user_id, amount, type, ref_id, created_at)`  
**后端**：`api/coins.js` — `/balance` `/transactions` `/tip`

### F-202：签到系统
**优先级**：P2  
**说明**：每日签到领币，连续签到奖励递增。  
**规则**：
| 连续天数 | 奖励 |
|---------|------|
| 第 1 天 | +5 |
| 第 2 天 | +8 |
| 第 3 天 | +12 |
| 第 4-6 天 | +15 |
| 第 7 天 | +30（周奖励） |
断签重置为第 1 天。漏签可用论坛币补签（50 币/天）。  
**数据表**：`sign_in_logs(user_id, signed_at, streak_days, coins_earned)`  
**前端**：签到日历组件  
**后端**：`api/signin.js`  
**依赖**：F-201

### F-203：徽章系统
**优先级**：P2  
**说明**：分两类——  
- **成就徽章**（自动颁发）：见 F-302  
- **商城徽章**（用户购买/管理员颁发）：`badges(id, name, desc, icon, type, price)`  
用户可佩戴展示 1-3 枚（个人信息页）  
**数据表**：`user_badges(user_id, badge_id, awarded_at, is_wearing)`  
**前端**：徽章墙、佩戴选择器  
**依赖**：F-201（购买用币）、F-302

## F-3XX 用户体系

### F-301：用户等级组
**优先级**：P1  
**说明**：**不同等级享受不同权限**，靠声望/论坛币/活跃度升级。  
**等级设计**：
| 等级 | 名称 | 升级条件 | 权限增益 |
|------|------|---------|---------|
| Lv0 | 新人 | 注册即得 | 可发帖/评论，每日 5 帖上限 |
| Lv1 | 学徒 | 声望≥50 | 无帖数限制 |
| Lv2 | 极客 | 声望≥200 | 可上传附件 |
| Lv3 | 黑客 | 声望≥500 | 自定义头像、发悬赏帖 |
| Lv4 | 大师 | 声望≥1500 | 可申请创建板块 |
| Lv5 | 传奇 | 声望≥5000 | 名字带特殊光效 |
**升级机制**：发帖+5 声望、被点赞+1、被采纳答案+20、被设精华+50。声望衰减：30 天无活动扣 10%  
**数据表**：`user_groups(user_id, level, exp, updated_at)`  
**前端**：等级徽章（带颜色：灰→青→紫→品红）、进度条  
**后端**：`api/level.js`  
**依赖**：F-001、F-101

### F-302：成就系统
**优先级**：P2  
**说明**：里程碑触发自动颁发徽章。  
**成就示例**：
- 「初来乍到」首次发帖
- 「话痨」累计 100 条评论
- 「人气王」单帖 100 赞
- 「富翁」论坛币 ≥1000
- 「元老」注册满 1 年
- 「破万」累计声望 10000
**数据表**：`achievements(id, code, name, desc, condition_json)`、`user_achievements(user_id, code, unlocked_at)`  
**触发**：在对应事件后检查并颁发  
**前端**：成就墙、解锁通知

### F-303：自定义头像
**优先级**：P2  
**说明**：默认头像是字母头像（按 `avatar_color`）。Lv3+ 可上传自定义头像。  
**存储**：R2 bucket `nexus-avatars`，原图 + 缩略图（Workers 内用 `@cf/serverless-ai` 或客户端 canvas 压缩）  
**校验**：jpg/png/webp，≤2MB，自动裁剪为正方形  
**前端**：头像上传 + 裁剪组件  
**依赖**：F-104（R2 基础设施）、F-301（等级限制）

### F-304：个人中心
**优先级**：P1  
**含**：主页（资料、等级、徽章、活动时间线）、资料编辑、我的帖子/评论/收藏  
**前端**：`profile.html`、`settings/profile.html`  
**后端**：`api/users.js`

## F-4XX 社区治理

### F-401：板块系统 + 用户申请建版
**优先级**：P2  
**说明**：管理员可建版；**用户（Lv4+）可申请创建板块**，需管理员审核通过。  
**含**：
- 板块 CRUD（管理员）
- 用户申请：填名称、简介、理由 → 进入审核队列
- 版块可设版主（管理员指派）
- 每个板块独立规则、图标、颜色（赛博风）
**数据表**：`boards(id, name, slug, desc, icon, color, owner_id, status)`、`board_applications(...)`  
**前端**：板块列表、板块申请表单、申请审核面板  
**后端**：`api/boards.js`  
**依赖**：F-301（等级门槛）、F-402（审核流程）

### F-402：内容审核系统（人工）
**优先级**：P0（合规要求）  
**说明**：管理员/版主可审核内容。  
**审核对象**：帖子、评论、板块申请、举报、头像  
**审核动作**：通过 / 拒绝 / 删除 / 隐藏 / 加精 / 置顶  
**数据表**：`moderation_queue(item_id, item_type, status, reviewer_id, action, note, created_at)`  
**前端**：审核后台（`/admin/moderation`）  
**后端**：`api/admin/moderation.js`

### F-403：AI 智能审核系统
**优先级**：P1（合规建议）  
**说明**：**可开关**。开启后，用户提交的所有内容（帖子/评论/板块申请/资料）先经 AI 预审核，再决定放行/标记/拦截。  
**对接**：通过统一适配层对接**多家大模型供应商**（OpenAI / Claude / 通义千问 / DeepSeek / GLM 等），管理员可在后台选模型、配 API Key。  
**审核维度**：政治敏感、涉黄、暴恐、辱骂、广告、个人信息泄露  
**处理策略**（可配置）：
| AI 判定 | 动作 |
|---------|------|
| 安全 | 直接发布 |
| 存疑 | 进入人工审核队列 |
| 违规 | 拦截，提示用户修改 |
| 严重违规 | 拦截 + 记录违规次数，多次则封号 |
**数据表**：`ai_review_logs(item_id, item_type, provider, model, verdict, score, reason, reviewed_at)`  
**配置**：后台开关、模型选择、阈值、API Key（`wrangler secret`）  
**后端**：`api/lib/ai-review.js`（统一适配器）、`api/admin/ai-config.js`  
**依赖**：F-402（结果汇入人工审核流）

### F-404：举报系统
**优先级**：P2  
**说明**：用户举报违规内容，进入审核队列。  
**数据表**：`reports(reporter_id, item_id, item_type, reason, status)`  
**前端**：内容旁「举报」按钮  
**依赖**：F-402

### F-405：禁言/封号
**优先级**：P1  
**说明**：管理员可对用户禁言（N 天）/ 永久封号，记录原因。  
**数据表**：`user_bans(user_id, type, reason, banned_until, banned_by)`  
**后端**：`api/admin/bans.js`

## F-5XX 账号安全

### F-501：换绑邮箱
**优先级**：P1  
**流程**：用户申请新邮箱 → 旧邮箱确认 → 新邮箱验证 → 完成换绑。每一步需 Turnstile。  
**后端**：`api/account.js` — `/change-email/request` `/change-email/confirm`  
**依赖**：F-001

### F-502：修改密码
**优先级**：P1  
**流程**：验证旧密码 → 输入新密码 → 更新。可选邮件二次确认。  
**后端**：`api/account.js` — `/change-password`

### F-503：密码找回
**优先级**：P1  
**流程**：填邮箱（+ Turnstile）→ 发重置链接（1 小时有效）→ 设新密码  
**复用**：`verification_tokens` 表，`type='password_reset'`  
**后端**：`api/auth.js` — `/forgot-password` `/reset-password`  
**依赖**：F-001

### F-504：账号注销
**优先级**：P1（合规要求，PIPL 规定用户有注销权）  
**流程**：用户申请 → 邮件二次确认 → 软删除（数据保留 90 天后真删）  
**注意**：注销前提示其内容归属处理选项  
**规范**：详见 `compliance.md`
