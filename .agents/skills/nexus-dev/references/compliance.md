# 法律合规规范（中国）🔒

NEXUS 面向中国大陆用户运营，受多重法律法规约束。**合规是上线的硬门槛，不是可选项。** 任何功能开发不得违背本规范。

## 重要声明

> ⚠️ 本文档由工程师编写，反映**工程实施层面的合规要求清单**，**不构成法律意见**。正式上线前，三份法律文档（《用户协议》《隐私政策》《社区规则》）的定稿**必须经执业律师审核**。本文档保证的是"技术框架满足合规底线"，不保证"内容措辞无法律风险"。

## 适用的主要法律法规

| 法律 | 简称 | 对项目的核心要求 |
|------|------|----------------|
| 《网络安全法》(2017) | 网安法 | 实名制、日志留存 6 个月、内容审核、安全事件报告 |
| 《数据安全法》(2021) | 数安法 | 数据分类分级、重要数据出境审批 |
| 《个人信息保护法》(2021) | **PIPL** | 知情同意、最小必要、用户权利（查询/更正/删除/注销）、个人信息出境 |
| 《互联网信息服务管理办法》 | — | ICP 备案、经营性 ICP 许可（如涉及收费） |
| 《互联网论坛社区服务管理规定》 | **直接适用** | 实名制、版主责任、先审后发、安全管理人员 |
| 《互联网用户账号信息管理规定》(2022) | — | 账号信息真实、不得冒用 |
| 《互联网跟帖评论服务管理规定》(2017修订) | — | 评论先审后发、评论审核员配备 |
| 《未成年人保护法》(网络保护章节) | 未保法 | 青少年模式、防沉迷、内容过滤 |
| 《生成式人工智能服务管理暂行办法》(2023) | **AIGC 办法** | AI 生成内容需显著标识、AI 服务需备案（如对外提供） |
| 《网络数据安全管理条例》(2024) | 网数条例 | 数据处理细节规范 |

## F-011：三份必备法律文档（注册前必须有）

注册表单的协议勾选框，必须可点击查阅以下三份文档。文档放在 `legal/` 目录，前端路由为 `/legal/terms.html` 等。

### 1. 《用户协议》（`legal/terms.html`）
**必备条款清单**（按监管要求）：
- [ ] 服务内容与性质说明
- [ ] 用户注册条件（**须满 18 周岁或监护人同意**，见未保法）
- [ ] **实名认证**条款（手机号实名，预留身份证号字段但非强制）
- [ ] 用户行为规范（**明确禁止的 9 类违法信息**，见下方清单）
- [ ] 知识产权归属（用户内容的授权范围）
- [ ] 账号管理（不得转让、出借、售卖）
- [ ] **账号注销**条款（链接到注销流程）
- [ ] 违约责任与封禁规则
- [ ] 争议解决（**管辖法院约定在运营方所在地**）
- [ ] 协议修改通知机制
- [ ] 联系方式（运营主体、客服、举报邮箱）

### 2. 《隐私政策》（`legal/privacy.html`）
**必备条款清单**（按 PIPL 第 17 条）：
- [ ] **个人信息处理者**名称及联系方式（运营公司主体）
- [ ] 收集的个人信息**目录**（逐项列明：手机号、邮箱、IP、设备标识、头像、内容等）
- [ ] 收集的**目的、方式、范围**（最小必要原则）
- [ ] **存储期限**（账号注销后留存期、日志 6 个月）
- [ ] **存储地点**（境内，不出境）
- [ ] 第三方共享/转让/公开（含 Resend 邮件、Cloudflare 服务、AI 审核供应商的**数据出境说明**——见下方专项）
- [ ] 用户权利行使方式（查询、复制、更正、删除、**注销**、撤回同意）
- [ ] 个人信息保护负责人联系方式（DPO）
- [ ] 跨境传输时的单独同意机制
- [ ] 自动化决策（AI 审核）的说明与退出权

### 3. 《社区规则》（`legal/rules.html`）
**必备**：
- [ ] 明确列出禁止发布的内容类型（9 类违法信息）
- [ ] 发帖/评论/板块申请的规则
- [ ] 违规处理阶梯（警告 → 删内容 → 禁言 → 封号）
- [ ] 申诉渠道

### 9 类违法信息（网信办规定，必须写进协议和 AI 审核维度）
1. 反对宪法所确定的基本原则
2. 危害国家安全，泄露国家秘密，颠覆国家政权，破坏国家统一
3. 损害国家荣誉和利益
4. 煽动民族仇恨、民族歧视，破坏民族团结
5. 破坏国家宗教政策，宣扬邪教和封建迷信
6. 散布谣言，扰乱社会秩序，破坏社会稳定
7. 散布淫秽、色情、赌博、暴力、凶杀、恐怖或者教唆犯罪
8. 侮辱或者诽谤他人，侵害他人合法权益
9. 含有法律、行政法规禁止的其他内容

## 实名制（网安法 + 论坛社区服务管理规定）

- **法律底线**：论坛社区类服务**必须配备用户实名信息**。
- **NEXUS 实现**：**管理员后台可控三档模式**（见 `admin-config.md`）：
  - `off`：不强制实名（初期冷启动可用，但需明确这**只是过渡**，正式运营后建议至少切到 `optional`）
  - `optional`：引导但不强制，未实名用户可浏览和基础互动，实名后解锁发帖/悬赏等高级功能
  - `required`：必须实名才能发帖/评论（**合规最安全档**，正式运营推荐）
- **实名方式**：**手机号 + 短信验证码**（阿里云短信 / 腾讯云短信，国内服务商）。
- **最小化原则**：只采集手机号（+验证），**不采集**身份证号/真实姓名（除非未来监管强制）。数据库只存手机号 SHA-256 哈希 + `phone_verified` 布尔标记。
- **预留扩展**：`users` 表预留 `id_card_hash` 字段（仅存 SHA-256，不存明文），若未来监管要求对接公安二要素核验，可启用。
- **隐私政策披露**：必须在《隐私政策》中写明：收集手机号的目的、存储方式、保留期限、用户权利。
- **注销时处理**：账号注销时，手机号哈希**同步删除**（PIPL"删除权"）。

## 第三方登录合规（PIPL + 个信规定）

### 通用合规要求
- **最小必要**：只取 `provider_uid`（标识）、昵称、头像、邮箱（如供应商提供）。**不**抓好友列表、仓库、动态等无关数据。
- **明示同意**：用户点击三方登录按钮 = 明示同意授权 NEXUS 读取其基本信息。《隐私政策》中必须逐项列出每个供应商收集的信息。
- **用户控制权**：用户可随时在「账号设置」中解绑三方账号（对应 PIPL"撤回同意权"）。
- **数据安全**：三方凭证（access_token）**用完即弃**，不存库。只持久化 `provider_uid` + 非敏感展示信息。

### QQ 登录专项
- **申请门槛**：需在 [connect.qq.com](https://connect.qq.com) 注册开发者，提交网站审核。需准备：网站备案号、隐私政策链接、ICP 证。
- **审核周期**：通常 1-3 个工作日。
- **用户须知披露**：隐私政策中说明 QQ 互联的授权范围（仅 `get_user_info`），并说明腾讯公司作为数据处理者的角色。
- **注意**：QQ 不返回用户邮箱，注册时需引导用户补充邮箱。

### Google 登录专项
- **数据出境**：Google 用户信息（openid、email、name、picture）属于**个人信息跨境传输**。按 PIPL 第 38-40 条：
  - 方案 A（推荐）：**默认关闭 Google 登录**（`oauth_google_enabled=0`），仅在面向海外用户时开启，并取得用户**单独同意**。
  - 方案 B：如必须开启，需做**数据出境安全评估**或签署标准合同（SCC），并在隐私政策中专项披露。
- **境内可达性**：Google OAuth 端点（accounts.google.com）在大陆需翻墙，用户体验差。**强烈建议默认关闭**，定位为海外用户可选。
- **OAuth 同意屏幕**：Google Cloud Console 配置时，"User data" 类别需如实填写数据用途。

### GitHub 登录专项
- **注册年限审核**（你的需求）：取 `created_at` 做是否满 1 年判断。**合规注意**：
  - 这是 NEXUS 自定的注册策略，非 GitHub 数据滥用，符合"最小必要"（只取一个时间字段用于防小号）。
  - 隐私政策中需说明：NEXUS 会读取 GitHub 账号创建时间，用于注册资格判断。
- **数据可达性**：GitHub API 通常可从大陆访问，无需翻墙，是**境内用户体验最好的三方选项**。

## 内容审核（先审后发 vs 先发后审）

### 法律要求
- 《论坛社区服务管理规定》《跟帖评论服务管理规定》要求**配备审核员**、**建立审核机制**。
- 对时政类等敏感内容，监管要求**先审后发**。

### NEXUS 审核策略（分级）
| 内容类型 | 策略 | 依据 |
|---------|------|------|
| 普通帖子/评论 | **先发后审**（AI 预审 + 人工巡查） | 用户体验优先 |
| 新注册用户前 3 条 | **先审后发** | 防垃圾注册 |
| 板块申请 | 先审后发 | 必须 |
| 被举报内容 | 立即复审 | 必须 |
| AI 判定"存疑/违规" | 进入人工队列（先审后发） | 见 F-403 |

### 日志留存（网安法第 21 条）
- 网络日志**至少留存 6 个月**。
- D1 表 `access_logs`、`operation_logs` 记录用户操作，定期归档。

## 未成年人保护（未保法）

- **注册年龄限制**：协议明确须满 18 周岁，未满须监护人同意。
- **青少年模式**（预留，建议尽早实现）：
  - 每日使用时长限制（40 分钟）
  - 每日 22:00-6:00 禁用
  - 内容过滤（仅展示适龄内容）
- **禁止采集**：不得收集 14 周岁以下儿童个人信息（如确需，须监护人单独同意）。

## 个人信息出境（PIPL 第 38-40 条 + 数据出境申报）

⚠️ **本项目的技术架构有数据出境问题，必须处理**：

| 数据流向 | 是否出境 | 处理 |
|---------|---------|------|
| 用户数据存 Cloudflare D1（全球边缘节点） | **可能出境** | 须在 Cloudflare 配置 **D1 数据本地化**（选 `APAC` 区域，含中国大陆），或迁移到境内服务商 |
| AI 审核调用海外模型（OpenAI/Claude） | **出境** | ① 优先用**国产模型**（通义/DeepSeek/GLM）；② 若用海外模型，须取得用户**单独同意**，并做**数据出境安全评估** |
| Resend 邮件服务（海外） | 邮箱地址出境 | 替换为境内邮件服务商（阿里云邮件推送/腾讯云 SES） |
| Cloudflare Turnstile | 设备指纹出境 | 评估后确认是否可接受；或换用国内验证码（极验/腾讯防水墙） |

**MVP 推荐方案**：
- D1 区域选 APAC
- AI 审核默认对接**国产模型**（GLM/通义/DeepSeek），海外模型列为可选
- 邮件用境内服务商
- Turnstile 保留，但隐私政策中披露

## 用户注销权（PIPL 第 47 条）

- 用户**有权注销账号**，平台**不得设置不合理条件**。
- 流程（F-504）：申请 → 邮件确认 → 软删除（`status='deleted'`，内容保留但隐藏）→ 90 天后真删。
- 注销前提示用户：内容归属处理（保留/删除/匿名化）。

## ICP 备案（硬性要求）

- 在中国大陆提供服务的网站**必须 ICP 备案**。
- 备案信息展示在**网站底部**（页脚），含备案号 + 公安备案号 + 实名核验链接。
- 备案主体：运营公司或个人（个人备案限制较多）。
- ⚠️ **Cloudflare 在国内访问需特殊处理**：要么用 Cloudflare 中国版（与京东云合作），要么国内用户走 CDN 加速。建议境内服务器 + 备案域名做主站，Cloudflare 仅做海外加速。

## 算法/AIGC 标识（AIGC 办法）

- AI 审核是**内部工具**，不对外提供 AIGC 服务，**无需 AIGC 备案**。
- 但若用户用 AI 生成内容发帖，应**显著标识**（"AI 生成"标签）。预留字段 `post.is_ai_generated`。

## 前端合规展示要点

- **注册表单**：协议勾选框默认**不勾**，未勾不可提交（法律要求"明示同意"）。
- **页脚**：必含 ICP 备案号、公安备案号、版权、运营主体、举报电话/邮箱。
- **敏感操作**（注销、换绑）：二次确认弹窗 + 邮件验证。
- **AI 生成内容**：发布时强制勾选"此内容由 AI 辅助生成"。
- **首次登录**：弹窗展示协议更新摘要（如有更新），需用户点"我已知悉"。
